Core aspects of the technology are:

  1. Property-Value Indexes
  2. Event Routers
  3. Storage
  4. Secure Networking
  5. Secure Identities

Property-Value Indexes

  • These indexes are basically huge lists/btrees of [unique id, type id, value] tuples for property::value pairs which provide a robust and scalable object/metadata model for collaborative infrastructures.

  • Basic indexes have been implemented by extending ZCatalog (especially its Plugin Indexes) and BTreeFolders. (Completed)

  • The Primary Indexes will be extended to cater for usage such as Latent Semantic Indexing. (To Be Implemented)

Event Routers

  • By integrating the Zope Application Server and Mod PubSub, which is the open sourced version of Knownow's Event Routing Platform, we have the necessary elements to make the whole infrastructure Event-Driven. (In Progress)

  • The event-driven network will allow interaction to be dynamic and in real time, and with integration with indexes, house the interface through which services are setup and provide capabilities such as Bi-directional Links and Sensors. (In Progress)

  • Further integration with the Openflow Engine will mean that we will have the ability to create extremely flexible and dynamic workflows to suit individual organisations. (To Be Implemented)

Storage

  • Currently all the data is stored directly into the Z Object Database (ZODB) which is a Python-specific object persistence framework that allows us to develop in pure Python and have the details of object persistence taken care of behind the scenes. (Completed)

  • By integrating the APE Object Mapping Framework, the data stored in the ZODB can be mapped onto all sorts of storage layers from the filesystem to relational databases and even to networks such as Freenet / Mnet / Bittorrent. (In Progress)

  • In the context of the civil society it also makes a lot of sense to maximise usage of underused resources such as hardware and thus a substrate such as Sharkypy's implementation of the Kademlia distributed hash table will be used to distribute the data storage. This will also enable an object oriented filesystem to access the data and facilitate the ability to transparently work online/offline. (To Be Implemented)

Secure Networking

  • Secure communications and link-to-link encryption between nodes are achieved by integrating Zope with the M2Crypto Toolkit which, in addition to providing cryptographic functions, also acts as a wrapper to the OpenSSL Library which implements Secure Sockets Layer (SSL) and Transport Layer Security (TLS). (Completed)

  • The data and the applications are distributed across node clusters using ZEO, ZSyncer and Twisted Spread. (In Progress)

  • Other aspects of Twisted Internet and Protocols will also be used in conjunction with the EGTP to provide further aspects of the network framework. CAKE may also play a role. (To Be Implemented)

Secure Identities

  • These are at the heart of this system, and use X.509 (through M2Crypto and pyCA) and OpenPGP to provide public key infrastructure (for digital signatures and encryption of confidential material) and a flexible identity management system with accountability, coherence, dependability and, above all, correct authentication. (In Progress)

  • The identities are also tied in with the Primary Indexes to define each individual's/organisation's Trust Matrix which lends itself to the evaluation in Reputation Servers with Accountability as well as usage of mechanisms such as Eigenvector Trust Metrics (In Progress)

  • Identities will be extended to include federated administration and access control systems such as Shibboleth and authentication mechanisms such as Twisted.Cred. CAKE may also play a role here. (To Be Implemented)

  • Combined with the distributed indexes and event routers, this provides the technical infrastructure needed for reputation-based points sytems, Capabilities Based Security, secure digital currency, Smart Contracts and trading systems. (To Be Implemented)
